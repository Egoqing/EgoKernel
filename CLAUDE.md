# EgoKernel - Claude 协作规则

## 项目概述

EgoKernel 是一个面向树莓派4B (ARM64架构) 的操作系统内核项目。这是一个**学习型项目**，目标是深入理解操作系统内核的原理与实现细节。

## 项目结构

## 编码执行规则

### 核心原则

**这是一个学习型项目，关键函数必须由用户亲自实现，不能完全自动生成。**

### 可自动生成的内容

以下内容可以由Claude自动生成，作为参考框架：

1. **头文件结构** - 函数声明、数据结构定义
2. **辅助函数** - 非核心的工具函数
3. **测试代码** - 单元测试和集成测试
4. **构建脚本** - Makefile、CMakeLists.txt
5. **注释和文档** - 代码注释、README
6. **调试代码** - printf风格的调试输出
7. **简单的初始化代码** - 标准化的初始化流程

### 必须由用户实现的关键内容

以下内容**必须留给用户实现**，Claude只能提供思路、框架和指导：

#### 1. 启动代码 (`boot/start.S`)
- ❌ CPU初始化序列
- ❌ 栈设置
- ❌ 异常向量表配置
- ❌ EL2到EL1的转换
- ❌ BSS段清零

#### 2. 内存管理 (`kernel/mm/`)
- ❌ 伙伴系统的分配/释放核心算法
- ❌ 页表映射/取消映射
- ❌ MMU启用序列
- ❌ Slab分配器的对象分配逻辑

#### 3. 中断处理 (`kernel/int/`)
- ❌ GIC-400的初始化序列
- ❌ IRQ分发逻辑
- ❌ 异常向量表实现
- ❌ 上下文保存/恢复汇编代码

#### 4. 进程管理 (`kernel/proc/`)
- ❌ 上下文切换汇编代码 (`switch.S`)
- ❌ 调度器选择算法
- ❌ 进程状态转换逻辑
- ❌ fork/exec的核心实现

#### 5. 设备驱动 (`kernel/drivers/`)
- ❌ UART的初始化序列
- ❌ GPIO的配置逻辑
- ❌ 定时器的配置和使用

### 协作模式

当用户请求实现某个功能时，Claude应该：

1. **分析需求** - 理解用户想实现什么
2. **提供思路** - 解释实现原理和关键步骤
3. **给出框架** - 提供代码骨架和接口定义
4. **指出关键点** - 标明哪些部分需要用户自己实现
5. **提供参考** - 给出类似的实现示例（其他内核）
6. **等待实现** - 用户完成关键函数后，帮助调试和完善

### 示例：正确的协作方式

**用户请求**: "帮我实现伙伴系统的分配函数"

**Claude的响应**:

```
伙伴系统分配函数的实现思路：

1. 查找匹配的order
2. 如果没有，从更大order分裂
3. 分裂算法：递归或循环
4. 更新页描述符状态
5. 返回分配的页面

关键函数签名（框架）：
```
page_t *buddy_alloc_pages(int order) {
    // TODO: 实现分配逻辑
    // 1. 从free_list[order]开始查找
    // 2. 如果找到空闲块，直接返回
    // 3. 如果没有，查找更大order并分裂
    // 4. 更新页描述符的state和order
}
```

实现提示：
- 使用list_empty()检查空闲链表
- 使用list_first_entry()获取第一个空闲块
- 分裂时需要递归调用buddy_split()
- 参考docs/memory.md中的算法描述
```

### 代码标记

在生成的代码中，使用以下标记明确指出用户需要实现的部分：

```c
// ===== USER IMPLEMENTATION REQUIRED =====
// 这是核心算法，请根据docs/memory.md中的说明实现
// ===== END USER IMPLEMENTATION =====
```

### 辅助程度划分

| 内容类型 | Claude行为 | 示例 |
|---------|-----------|------|
| 核心算法 | 提供框架和指导 | 伙伴系统分配逻辑 |
| 汇编代码 | 提供寄存器清单和流程 | 上下文切换 |
| 硬件操作 | 提供寄存器地址和位定义 | GIC初始化 |
| 接口定义 | 完整实现 | 头文件中的函数声明 |
| 工具函数 | 可以完整实现 | 字符串函数 |
| 测试代码 | 完整实现 | 单元测试 |

### 调试支持

当用户实现的关键函数有问题时：

1. **分析错误** - 帮助定位问题所在
2. **提供调试思路** - 建议添加哪些打印语句
3. **解释原理** - 说明为什么会出现问题
4. **给出版本** - 提供修复思路，但不是完整代码

### 学习资源

在提供指导时，可以引用：

- `docs/` 目录下的设计文档
- Linux内核源码（相关部分）
- ARM官方文档
- 其他教育型OS项目（如xv6, rpi-os）

### 禁止事项

1. **不要** 直接给出关键函数的完整实现
2. **不要** 替代用户思考和设计
3. **不要** 绕过学习过程提供"捷径"
4. **不要** 使用过于高级的抽象掩盖底层细节

### 鼓励事项

1. **鼓励** 用户理解原理后再实现
2. **鼓励** 用户自己查阅硬件文档
3. **鼓励** 用户阅读参考代码
4. **鼓励** 用户通过实践加深理解

## 配置说明

### 目标平台
- 树莓派4B (BCM2711)
- 四核 ARM Cortex-A72
- ARMv8-A 架构

### 编译环境
- `aarch64-none-elf-gcc`
- 无标准库支持 (ffreestanding)

### 内存配置
- 内核加载地址: 0x80000
- 页大小: 4KB
- 内核栈: 16KB
- 目标RAM: 1GB
